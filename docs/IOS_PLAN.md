# Thoth iOS Development Plan

## Overview

**Purpose**: Add native iOS support to Thoth via Capacitor — same React codebase, native iOS App Store app.

> Thoth is a cross-platform dream journal. Capacitor means zero iOS-specific frontend code. This plan covers the native iOS shell setup, Xcode configuration, and CI/CD pipeline changes.

---

## How Capacitor iOS Works

Capacitor wraps the web app in a native iOS shell (WKWebView). No Swift/Kotlin rewrite needed for the app logic.

```
React Code (src/)
    ↓ vite build
HTML/JS/CSS (dist/)
    ↓ npx cap sync ios
Native iOS Shell (ios/)
    ↓ xcodebuild
.thoth.ipa  ← App Store / TestFlight distribution
```

**Key difference from Android**: The `ios/` folder is auto-generated by Capacitor and should **not** be manually edited — all customization goes through `capacitor.config.ts` and the web layer.

```typescript

  contentInset: 'automatic',      // Safe area handling
  backgroundColor: '#0a0a0f',     // Match app dark theme

    launchShowDuration: 2000,
    backgroundColor: '#0a0a0f',
    showSpinner: false,

  StatusBar: {
    style: 'DARK',

  },

```

## Xcode Configuration (Manual — One-Time Setup)

### Signing

- **Team ID**: Required (Apple Developer account)
- **Bundle Identifier**: `com.thoth.dreamarchive`
- **Minimum iOS Version**: iOS 14.0 (Capacitor 8 requirement)

### Capabilities (enable in Xcode)

1. **Push Notifications** — for local dream reminders
2. **Background Modes** → **Audio** — recording in background
3. **App Groups** — `group.com.thoth.dreamarchive` (for future Watch app)

### Info.plist Additions

```xml
<!-- Microphone -->
<key>NSMicrophoneUsageDescription</key>
<string>Thoth needs microphone access to record your dreams.</string>

<!-- Location (optional — for global dream map) -->
<key>NSLocationWhenInUseUsageDescription</key>
<string>Add your location to the global dream map (anonymous).</string>

<!-- Background Audio -->
<key>UIBackgroundModes</key>
<array>
    <string>audio</string>
</array>
```

### App Icon

- Generate from the existing design (1024×1024 source)
- Use Capacitor's auto-generated asset pipeline

## CI/CD Changes

### Current State

`release.yml` only builds Android on a `ubuntu-latest` runner.

### Target State

```yaml
# release.yml — dual platform build
jobs:
  ci:
    runs-on: ubuntu-latest          # Linux: lint + test
    steps: [...]

  build:
    runs-on: ubuntu-latest
    steps: [web build, sync, ...]

  build-android:
    runs-on: ubuntu-latest         # Linux runner OK
    needs: build
    steps: [assembleDebug, upload artifact, ...]

  build-ios:
    runs-on: macos-latest          # macOS required
    needs: build
    steps:
      - actions/checkout
      - actions/setup-node
      - run: npm ci && npm run build
      - run: npx cap sync ios
      - name: Setup Code Signing
        uses: apple-actions/import-codesign-certs@v2
        with:
          p12-cert-data: ${{ secrets.IOS_P12_CERTIFICATE_BASE64 }}
          p12-password: ${{ secrets.IOS_P12_PASSWORD }}
      - name: Import provisioning profile
        run: |
          echo '${{ secrets.IOS_PROVISIONING_PROFILE }}' | base64 -d -o App.mobileprovision
      - name: Build iOS
        run: |
          xcodebuild -workspace ios/App/App.xcworkspace \
            -scheme App \
            -configuration Release \
            -sdk iphoneos \
            CODE_SIGN_STYLE=Manual \
            CODE_SIGN_IDENTITY="${{ secrets.IOS_CODE_SIGN_IDENTITY }}" \
            PROVISIONING_PROFILE_SPECIFIER="${{ secrets.IOS_PROVISIONING_PROFILE_NAME }}" \
            DEVELOPMENT_TEAM="${{ secrets.IOS_TEAM_ID }}" \
            -derivedDataPath build \
            -archivePath build/App.xcarchive \
            archive
      - name: Export .ipa
        run: |
          xcodebuild -exportArchive \
            -archivePath build/App.xcarchive \
            -exportOptionsPlist ios/App/exportOptions.plist \
            -exportPath dist
      - name: Upload .ipa artifact
        uses: actions/upload-artifact@v4
        with:
          name: thoth-ios-${{ github.ref_name }}
          path: dist/App.ipa
```

### GitHub Secrets Required

| Secret | Description |
|--------|-------------|
| `IOS_P12_CERTIFICATE_BASE64` | Base64-encoded .p12 code signing certificate |
| `IOS_P12_PASSWORD` | Password for the .p12 file |
| `IOS_PROVISIONING_PROFILE` | Base64-encoded .mobileprovision file |
| `IOS_CODE_SIGN_IDENTITY` | e.g. `Apple Distribution: Name (TEAM_ID)` |
| `IOS_PROVISIONING_PROFILE_NAME` | e.g. `Thoth App Store` |
| `IOS_TEAM_ID` | Your Apple Developer Team ID |

> **Note**: For TestFlight/internal testing, use `CODE_SIGN_IDENTITY="Apple Distribution"`. For App Store, use `APPSTORE` flavor provisioning.

## Development Workflow

### Local Setup (macOS only)

```bash
# 1. Clone repo
git clone https://github.com/ceaserzhao/ThothApp.git
cd ThothApp

# 2. Install dependencies
npm install
npx cap sync ios

# 3. Open in Xcode
open ios/App/App.xcworkspace

# 4. In Xcode:
#    - Select a simulator or device
#    - Set your Team in Signing & Capabilities
#    - Hit Run (Cmd+R)
```

### Updating iOS After Web Changes

```bash
npm run build          # Rebuild web app
npx cap sync ios       # Sync to iOS native shell
open ios/App/App.xcworkspace  # Re-open Xcode
# Hit Run again
```

## iOS vs Android Comparison

| Aspect | iOS | Android |
|--------|-----|---------|
| Setup command | `npx cap add ios` | Already done |
| Build runner | `macos-latest` | `ubuntu-latest` |
| Build tool | xcodebuild | Gradle |
| Output artifact | `.ipa` | `.apk` |
| Signing | .p12 + .mobileprovision | keystore |
| App Store | App Store Connect | Play Console |
| Code signing CI | apple-actions/import-codesign-certs | Native Gradle |
| Local dev requires | macOS + Xcode | Any OS + Android Studio |

## Key Risks

| Risk | Mitigation |
|------|-----------|
| No Mac → can't run Xcode locally | CI can still build (macOS runner); local dev needs Mac |
| Apple Developer account required ($99/yr) | Required for real device + App Store |
| Code signing expires annually | Renew .p12 and .mobileprovision before expiry |
| App Store rejection (guidelines) | Review Apple HIG before submission |
| iOS simulator ≠ real device | Test on real iPhone before release |
